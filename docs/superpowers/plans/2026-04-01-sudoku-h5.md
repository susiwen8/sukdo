# Sudoku H5 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an independent mobile-first H5 Sudoku game page and solver page that reuse `miniapps/shared` logic in the browser.

**Architecture:** Introduce a lightweight browser CommonJS runtime that loads `miniapps/shared` modules directly from static files, then build two H5 page shells on top: one game page and one solver page. Keep the existing desktop web pages and native mini-app pages intact, and verify the new H5 layer through focused file, runtime, and DOM-helper tests plus the existing shared regression tests.

**Tech Stack:** Static HTML/CSS, browser ESM, lightweight in-browser CommonJS loader, existing `miniapps/shared` CommonJS modules, Node built-in test runner
